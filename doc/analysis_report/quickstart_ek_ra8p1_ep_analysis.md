## 1. 概要

### 1.1 プロジェクトの目的
Renesas EK-RA8P1評価ボード向けの拡張ペリフェラルデモプロジェクト。OV5640カメラモジュールによるリアルタイム映像キャプチャ、タッチパネル入力、マルチスレッド制御を実現する総合的なリファレンス実装。

### 1.2 ターゲットボード・MCU
| 項目 | 仕様 |
|------|------|
| ボード | Renesas EK-RA8P1 |
| MCU | RA8P1（デュアルコア） |
| メインコア | ARM Cortex-M85 @ 1GHz |
| サブコア | ARM Cortex-M33 @ 250MHz |
| 内蔵RAM | 1.87MB |
| 外部SDRAM | 128MB |
| FSPバージョン | 6.2.0 |

### 1.3 使用フレームワーク・ライブラリ
- **Renesas FSP (Flexible Software Platform)** 6.2.0
- **FreeRTOS** - リアルタイムOS
- **D/AVE 2D** - ハードウェアグラフィックスアクセラレータ
- **MIPI CSI-2** - カメラインターフェース
- **VIN** - ビデオ入力モジュール
- **ツールチェーン**: LLVM Embedded Toolchain for Arm

## 9. セキュリティ観点

### 9.1 バッファオーバーフローリスク

#### カメラデータ処理
| 項目 | 値 |
|------|-----|
| IMAGE_WIDTH | 1024 |
| IMAGE_HEIGHT | 600 |
| BYTES_PER_PIXEL | 2（RGB565） |
| フレームバッファサイズ | 1,228,800 bytes（1.2MB） |
| パディング | 20,480 bytes |

**リスク評価**:
- `display_next_buffer_set()`でポインタ直接代入（境界チェックなし）
- VINコールバックの`p_args->p_buffer`は検証なしで渡される
- **緩和策**: バッファは静的に確保済み、サイズは固定

#### タッチ入力処理
- タッチポイント配列: `st_touch_coords_t { uint16_t x[5], uint16_t y[5] }`
- `process_menu_detection()`でハードコードされたヒットボックス比較
- **リスク**: 座標値の範囲チェックなし（0xFFFFの可能性）
- **緩和策**: ヒットボックスは画面境界内に制限（0-1024, 0-600）

### 9.2 ハードコードされた認証情報

**検出された固定値**:
| 項目 | 値 | リスク |
|------|-----|--------|
| OV5640 I2Cアドレス | 0x3C | デバイス固有（問題なし） |
| FT5316 I2Cアドレス | 0x38 | デバイス固有（問題なし） |
| GPIOピン定義 | BSP_IO_PORT_* | ハードウェア固有（問題なし） |

**機密キーの検出**: なし（デモ/リファレンス実装のため）

### 9.3 入力検証の問題

#### カメラレジスタアクセス
```c
void ov5640_write_reg(uint16_t reg, uint8_t dat) {
    wr_sensor_reg16_8(&g_board_i2c_master_ctrl, reg, temp);
    read_back = ov5640_read_reg(reg);
    if(read_back != dat) { /* サイレント失敗（BKPTコメントアウト） */ }
}
```
- **問題**: 任意のレジスタアドレスを受け入れ（範囲検証なし）
- **リスク**: ファームウェア改変で任意レジスタへの書き込み可能
- **緩和策**: 信頼されたハードウェア上でのみ実行

#### タッチパネルI2C
```c
fsp_err_t read_reg8(uint8_t reg_address, uint8_t *reg_val, ..., uint8_t bytes) {
    R_IIC_MASTER_Write(..., &reg_address, bytes, true);
    R_IIC_MASTER_Read(..., reg_val, bytes, false);
}
```
- **問題**: `bytes`パラメータがreg_valバッファサイズより大きい場合、範囲外アクセス
- **リスク**: 呼び出し元が不正なバッファサイズを渡した場合のスタック破壊
- **緩和策**: 呼び出し箇所では`0x01`がハードコード

### 9.4 メモリ安全性

#### SDRAMバッファ配置
- `uint8_t SDRAM_pixel_data[1024 * 600 * 4 + 1]` - ローカル変数として宣言
- 実際は`.sdram`セクション属性でSDRAMに配置
- **リスク**: 見かけ上スタックオーバーフロー（実際はリンカセクションで安全）

#### ISRからのセマフォ
```c
void irq19_tp_callback(...) {
    xSemaphoreGiveFromISR(g_irq_binary_semaphore, &xHigherPriorityTaskWoken);
}
```
- **正常**: FromISRバージョンのFreeRTOS APIを使用
- **リスクなし**: 適切な同期プリミティブ

### 9.5 セキュリティ改善推奨

1. **レジスタアドレス検証追加**: OV5640読み書き関数で有効範囲チェック
2. **タッチ座標範囲検証**: ヒットボックス計算前に座標値検証
3. **バッファサイズチェック**: I2C読み書き関数でサイズ検証追加
4. **ウォッチドッグタイマー**: カメラ初期化タイムアウト監視
5. **CRC/チェックサム**: 重要なディスプレイコマンドに追加
6. **タッチIRQレート制限**: DoS防止のための処理レート制限
